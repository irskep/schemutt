Schemutt parses XML documents and displays a high-level view of their structure.

Imagine that you are asked to parse a file like this without any documentation:

simple_example.xml (full version)

While it might be feasible to go through the file by hand and figure out the ranges of values of each field and how they correlate, Schemutt can take away some of the pain:

You can easily see how the tags are nested, what their attributes are called, and what their attributes' typical values are. (The number of example values shown is based on the --line-length switch.)

Schemutt can be instructed to correlate the values of the attributes with the --find-foreign-keys flag, which will identify pairs of attributes that share more than 10% of their values:
